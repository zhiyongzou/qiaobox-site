# 巧匣：Markdown 转微信公众号神器，图片、代码块和排版一键搞定

> 如果你经常用 Markdown 写公众号文章，大概率遇到过同一个问题：写作很顺，发布很烦。

Markdown 足够简单，但当你要把 Markdown 转成微信公众号格式时，麻烦就来了。真正耗时间的往往不是写完正文，而是发布前那段返工：

```
* 本地图片要不要先传图床，或者再传一遍微信图片库？
* 复制到公众号后台后，图片会不会丢，样式会不会散？
* 代码块、引用、表格、行内代码还要不要重新调？
* Mermaid 图能不能跟着文章一起进入公众号草稿？
* 换一套公众号排版风格，是不是又要改正文？
* 发布之外，能不能顺手导出 PDF、HTML 或图片留档？
```

**巧匣** 就是为这段流程做的 Mac 应用。

它不是只帮你“打开一个 Markdown 文件”，而是把写作、预览、微信公众号排版、代码展示、图文管理、公众号格式复制和多格式导出放到同一个工作台里。写完后直接复制公众号格式，本地图片(**不需要先把每张图片上传到图床**)、代码块和已渲染的 Mermaid 图可以一起带到剪贴板里。

## 用巧匣前后，发布流程有什么变化

如果不用专门的 Markdown 转微信公众号工具，一篇文章写完后，通常还要做这些事：

- 把本地图片一张张上传，再回到正文里确认位置；
- 重新检查代码块、引用、表格和行内代码的样式；
- 把 Mermaid 图单独截图或导出，再插回公众号草稿；
- 复制到公众号后台后，从头到尾预览一遍，检查有没有格式散掉。

用巧匣之后，流程会更短：

- 在本地继续用 Markdown 写原文；
- 在预览里确认标题、图片、代码块和 Mermaid 图；
- 直接复制公众号格式；
- 粘贴到微信公众号后台，重点检查内容和最终效果。

它不会让公众号后台消失，但会尽量减少“写完之后又重排一遍”的重复劳动。

## 预览，不只是“能看见”

很多 Markdown 工具的预览，只是把正文转成 HTML。

这当然有用，但对要发布的人来说还不够。你真正关心的是：读者看到这篇文章时，是否清楚、舒服、有重点。

巧匣内置多套预览主题。写技术文章时，你可以选择接近 GitHub 的清爽样式，让代码和层级更干净；写公众号文章时，也可以切换到更适合移动端阅读的排版风格，比如渐变风、卡片风、杂志专栏风、科技蓝绿风或暖色手账风。

同一份 Markdown 内容，可以根据发布场景切换不同视觉表达。

技术教程强调代码清晰，经验分享强调阅读节奏，产品介绍强调重点突出。你不需要为了换风格去改正文，也不需要为不同平台维护好几份内容。

## 技术内容，也能排得体面

如果你经常写开发教程、知识笔记或产品文档，最容易破坏版面的通常不是正文，而是代码和图示。

代码块太长，读起来费劲；高亮不清楚，重点看不出来；流程图用截图保存，后面想改又很麻烦。

巧匣对这些技术写作场景做了专门支持：

- 代码块支持高亮，关键字、字符串和注释更容易区分；
- 长代码以更适合阅读的方式展示，不会把正文节奏挤乱；
- Mermaid 流程图、结构图可以直接在预览区渲染，复制公众号格式时也能转成图片；
- 本地图片可以跟随文章一起组织，预览、复制和导出都更稳定。

这对写教程尤其有用。

你可以继续用 Markdown 保存可维护的原文，用 Mermaid 保存可修改的图示，用本地图片保存必要截图。文章不是一堆临时复制出来的素材，而是一份可以长期整理、迁移和复用的内容资产。

## 发布之外，也要方便留档

内容不应该只存在于一个平台里。

一篇文章写完后，你可能要发公众号，也可能要导出 PDF 发给同事，保存 HTML 进资料库，或者生成图片用于分享。

巧匣支持导出 HTML、PDF、PNG 和公众号格式。同一份 Markdown 草稿，可以根据不同场景生成不同形态：

- 发公众号：复制适配后的格式，本地图片和 Mermaid 图一起带上，减少后台返工；
- 做归档：导出 HTML 或 PDF，方便保存和转发；
- 做分享：导出 PNG，用于长图、课程资料或内部沟通；
- 做长期管理：保留原始 Markdown 文件，方便多年后继续整理。

平台会变，分发渠道会变，但原始内容最好始终掌握在自己手里。

Markdown 文件轻量、清晰、容易备份，也方便迁移。巧匣做的事情，就是让这份原始内容更容易变成不同场景需要的成品。

## macOS 安装提醒：遇到安全提示怎么办

目前巧匣提供的是 macOS 编译包，不是 App Store 版本。

第一次打开时，macOS Gatekeeper 可能会提示类似下面的信息：

- “无法打开‘巧匣’，因为无法验证开发者。”
- “‘巧匣’无法打开，因为 Apple 无法检查其是否包含恶意软件。”

这是 macOS 对非 App Store、未完成公证分发的应用做的安全提醒，并不等于 App 一定有问题。你只需要确认安装包来源可信，再手动放行一次。

安装前建议先确认三件事：

- 安装包来自官方渠道或可信联系人；
- 文件名、App 名称和图标都和“巧匣”一致；
- 安装过程中没有要求你输入 Apple ID 密码，也没有要求安装陌生描述文件。

如果来源说不清楚，或者文件名、图标明显对不上，就不要继续安装。

正常安装可以按这个流程：

- 如果收到的是 `.zip` 文件，先双击解压；
- 将 `巧匣.app` 拖到“应用程序”文件夹；
- 第一次打开如果被 macOS 拦截，进入“系统设置” -> “隐私与安全性”；
- 在“安全性”区域找到关于巧匣的提示，点击“仍要打开”或“Open Anyway”；
- 再次打开巧匣，确认启动正常。

系统可能会要求你输入 Mac 登录密码，或使用 Touch ID 确认。这是 macOS 的本机安全验证，不是 Apple ID 密码。

打开后，可以先新建一篇 Markdown，输入标题、列表和代码块，确认左侧编辑、右侧预览是否正常。

接着试一下这几个核心功能：

- 切换预览主题，看看文章在不同风格下的效果；
- 插入一张本地图片，确认图文预览正常；
- 写一小段代码块或 Mermaid 图，检查高亮和图示效果；
- 使用公众号格式复制，粘贴到公众号后台草稿里，确认图片、代码块和排版一起带过去；
- 导出 PDF、HTML 或 PNG，确认文件可以正常打开。

如果系统询问文件、照片或网络访问权限，可以根据实际使用场景选择。暂时用不到的权限，可以先不授权，后面需要时再到系统设置里调整。

如果只是临时体验，测试完可以直接删除 `巧匣.app`。巧匣不会要求你安装描述文件，也不需要额外信任开发者证书。

## 适合谁使用

如果你只是偶尔写几行备忘，普通文本编辑器就够了。

但如果你经常写公众号文章、技术教程、产品说明、课程资料、项目复盘，或者希望把 Markdown 当作长期内容资产来管理，巧匣会更适合你。

它适合这些场景：

- 写作时想保持 Markdown 的专注感；
- 发布前希望看到更接近成品的预览效果；
- 技术文章里经常有代码块、流程图和本地图片；
- 希望本地图片不用先传图床，就能随公众号格式一起复制；
- 需要把文章复制到公众号后台，减少图片和格式返工；
- 希望同一份 Markdown 能导出 PDF、HTML、PNG 等多种格式。

简单来说，**巧匣更适合把写作当成长期工作流的人**。

它不只是一个 Markdown 查看器，而是一套从起草、预览、排版到导出发布的内容工作台。

如果你正在找一款 Markdown 转微信公众号工具，可以先拿一篇真实草稿试试 **巧匣**：插入一张本地图片，放一段代码块，再加一个 Mermaid 图，然后复制到微信公众号后台，看图片、代码块和排版能不能一起带过去。
